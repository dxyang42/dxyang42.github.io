Token 焦虑：AI 从包月到按量，我还舍得用吗

发表于2026-07-20|更新于2026-07-20|随笔
|浏览量:

大模型行业今年集体涨价，我观望了半年，最近轮到自己做决定了。

先把时间线摆出来，以 Kimi 为例。

1 月，Kimi 把编程订阅从按请求次数改成按 token 计费。写一行代码和写一千行代码，消耗天差地别，重度用户当场体感翻倍。

过了几个月，又改了一次。原来各功能额度独立，Agent、PPT、深度研究各用各的，互不挤占。改完之后，除了 Kimi Code 单独一个池，其余功能合并共享一个 token 池。官方说法是额度变多了、更灵活了。也有付费用户反映，新规则下额度消耗变快了，偶尔还会遇到没说清原因的使用限制。这些说法我没法逐一核实，但社区里的讨论热度是实打实的。

7 月 17 日，K3 发布。模型确实强，但 API 输入价格涨到上一代的三倍，输出快四倍。

也不止 Kimi。智谱一年涨了三次价，MiniMax 改了计费口径，阿里云停了便宜套餐，DeepSeek 搞峰谷定价。行业的意思很明白了，补贴期结束，开始算账。

我的处境是这样。我是 AI agent 重度用户，一天大概烧 10M tokens。Kimi 订阅下个月到期，我不打算续了。涨价是一个原因，更重要的是花钱买来的额度，权益规则对方说改就改。这买卖不划算。

不续订，就得按量付费。纠结跟着来了。

第一重纠结：按量真的便宜吗

算账。一天 10M tokens，一个月 3 亿。agent 流量的大头是输入，按九成输入、一成输出估。

方案 单价 月成本估算
Kimi 订阅 199 元封顶 199 元，但权益规则说变就变
DeepSeek V4 Pro 官方按量 输入 3 元/M，缓存命中 0.025 元/M，输出 6 元/M 250 到 350 元
中转站的 GLM coding 模型 输入 0.06 美元/M，输出 0.22 美元/M 约 160 元
中转站免费档（GLM、MiniMax 都有 free 版） 0 0 元，代价是限速排队

结论很直白，DeepSeek 按量在我这个用量下，并不比订阅便宜。也正常，订阅本来就是批发价，量大从优。3 亿 tokens 一个月，按量就是给订阅打工。

可能有人会问，为什么不直接买 GLM 官方的 Coding Plan。两个原因。一是没有多模态，看个图、读个截图就抓瞎。二是常年缺货，想买都买不到。它取代不了 Kimi Coding Plan 这种什么都能干的订阅，所以中转站按量才是现实路径。

第二重纠结，也是真正折磨我的

按量之后，我还舍得这么用吗？

订阅的时候，一天 10M tokens 烧起来毫无心理负担。月费付完了，每多用一 token 的边际体感是零。改成按量，每一次调用都在花钱。我会不会下意识省着用？省着省着，生产力不就掉了？

后来我想通了。这个“舍得”，本来就是幻觉。

10M 一天，里面有多少是真需求，多少是反正不心疼的浪费？订阅制让我从来没动力去分辨。按量之后，每一次调用都被定价，配不上价值的消耗会被我自己筛掉。

这逻辑耳熟。我在《给时间定个价》里写过一模一样的，工作时间被明确定价之后，配不上价值的事会被资本筛掉。现在轮到我自己当这个资本。

而且真实成本比直觉低得多。agent 流量的大头是重复上下文，每轮对话都要把历史重新发给模型。这部分走缓存，DeepSeek 缓存命中 0.025 元/M，约等于不要钱。真正花钱的只有新输入和输出。账面 10M，有效账单可能只有三分之一。

剩下的是心态，心态有药

给自己设月度预算，比如 150 元，当成订阅来花。预算内放开用，花超了下个月再加。相当于自己给自己发订阅，区别是规则我定。

该重的地方别省。复杂编程、长文档分析这种真正产出价值的场景，继续用贵模型。省的是闲聊和格式化这种杂活，全丢免费档。生产力不会掉，掉的是浪费。

真要出现为了省钱不敢用的苗头，说明预算定低了，加钱就是。怎么都比把钱交给一个规则说变就变的平台强。

最后

补贴时代结束了，我说不好这是好事还是坏事。但有一点我确定，免费的额度让人懒惰，明码标价让人清醒。工具还是那些工具，差别在你从随便用变成想清楚再用。

想清楚再用，大概就是行业挤泡沫的方式。

也是我挤自己水分的方式。

顺带一提，写完这篇博客，我的 agent 正好烧了 6M token。按缓存价折算，成本不到两块钱。
